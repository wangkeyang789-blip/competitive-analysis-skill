# 竞品分析报告生成器

生成专业的、结构化的竞品分析报告，覆盖竞品选择、多维度分析和可执行的结论建议。

## 工作流程

1. 接收用户想要分析的产品/行业
2. 确定分析模式：
   - **自动模式**：用户给产品/行业名称 → 用 WebSearch 收集竞品和市场信息
   - **资料模式**：用户自己提供调研材料 → 基于已有资料整理
   - **混合模式**：用户提供部分资料，需要搜索补充
3. 向用户确认：分析范围（哪些竞品）、重点关注的维度
4. 按下面模板生成完整报告
5. 输出 Markdown（对话中展示）+ 保存为 Word 文档（.docx）

## 报告模板

严格按照以下结构生成报告：

### 1. 竞品介绍
- 市场背景：行业现状、用户痛点、市场趋势
- 本次分析涉及的主要竞品
- 竞品分析目的（要点列表）

### 2. 竞品选择
- 为什么选择这些竞品（市场份额、用户规模、相关性）
- 支撑选择的数据或证据（搜索结果、行业报告）

### 3. 竞品分析

#### 3.1 竞品介绍
每个竞品需要包含：
- 成立时间
- 平台模式（C2C/B2C/B2B 等）
- Slogan
- 产品介绍（核心功能，要点列表）
- 商业模式（收入来源）

#### 3.2 产品定位分析
对比分析各竞品的：
- 目标用户群体
- 核心价值主张
- 市场定位策略
- 品牌认知

#### 3.3 商业策略分析
对比：
- 收入模式和定价
- 增长策略
- 合作/分发渠道
- 成本结构

#### 3.4 产品设计分析
对比：
- 核心功能
- 交互/用户体验思路
- 关键产品决策
- 技术差异化（如果能找到相关信息）

#### 3.5 产品数据分析
包含可获取的指标：
- 用户规模（如果能找到 MAU/DAU）
- 市场份额
- 增长趋势
- 任何公开可见的表现数据

#### 3.6 用户反馈分析
- 用户评价倾向（应用商店、社交媒体、公开论坛）
- 主要好评点
- 主要吐槽点
- 功能诉求

#### 3.7 竞品总结
- 各维度对比汇总
- 关键差异和模式
- 竞品行为反映的行业趋势

### 4. 结论和建议

#### 4.1 启发点
- 每个竞品值得学习的地方
- 行业最佳实践
- 值得借鉴的模式

#### 4.2 机会点
- 市场空白或未满足的用户需求
- 竞品的薄弱环节
- 可差异化的方向
- 建议的后续行动

## 数据来源规则

- **自动模式**：用 WebSearch 为每个章节搜集信息。搜索竞品背景、市场数据、用户评价、产品功能
- **可信度标注**：没有明确来源的数据，添加置信度标识，如"（据公开资料）"、"（估算）"等
- **每个竞品**在自动模式下至少需要 2-3 个不同来源的信息
- 如果某个章节搜索不到足够信息，明确标注而非编造

## 输出格式

### Markdown
- 完整报告在对话中展示
- 按层级使用标题（# → ## → ### → ####）
- 对比章节使用表格
- 功能和详情使用要点列表

### Word 文档（.docx）
- 保存到 `competitive-analysis/[产品名]-竞品分析.docx`
- 先把 Markdown 报告保存为临时 `.md` 文件
- 用内置脚本转换：`python scripts/markdown_to_docx.py <input.md> <output.docx> --title "报告标题"`
- 如果默认 `python` 不可用，尝试 `/c/Users/USER/miniconda3/python.exe`
- 保留标题层级、表格和列表

## 质量检查清单

输出前确认：
- [ ] 所有模板章节都有内容（数据不足处标注了缺口）
- [ ] 每个竞品的覆盖维度一致
- [ ] 论断有搜索依据或明确标注为估算
- [ ] 结论和建议具体可执行，不是泛泛而谈
- [ ] Markdown 格式正确
- [ ] Word 文档生成成功并已保存
